rocket_launchInicio rapido

Acepta pagos con
MedianetPay en minutos.

Una sola llamada a nuestra API genera un link de pago seguro. El cliente paga en la pagina certificada PCI DSS de MedianetPay, y tu recibes la confirmacion en tiempo real via webhook.

1

Obtener credenciales

Crea tu cuenta en medianetpay.ec y genera tu key_webservice desde Configuracion → Seguridad.

2

Crear el link de pago

Llama al endpoint WebCheckout con el monto y descripcion. Recibiras un link de pago. Redirige al cliente a ese link.

3

Recibir confirmacion

MedianetPay hace un POST a tu url_back con el resultado. Actualiza el estado de la orden.

Ejemplo minimo en Python

Con estas 15 lineas tienes un flujo de pago completo funcionando en QA.

}

# -> redirigir al cliente a `link`
keyAutenticacion

Credenciales

MedianetPay no usa headers Authorization. Las credenciales van dentro del body JSON de cada request.

username Requerido

Tu correo de acceso a la plataforma MedianetPay. Maximo 30 caracteres.

key_webservice Requerido

Llave de 36 caracteres. Generala desde tu portal: Configuracion general → Seguridad Webservice → Llave Webservice. Esta llave se invalida si la regeneras.

warning Nunca expongas key_webservice en frontend publico (JS del navegador). Solo llamadas server-side.

paymentsCrear un pago

POST /app/webservice/webcheckout/rest

Genera un link de pago unico. El cliente es redirigido a ese link para completar el pago en la pagina certificada PCI DSS de MedianetPay.

POST https://api.medianetpay.ec:4443/app/webservice/webcheckout/rest

El servidor responde en menos de 2 segundos con el link de pago. Redirige inmediatamente al cliente a ese URL.

IVA Ecuador: El IVA vigente es 15%. El campo value_base_not_iva es el monto sin IVA.

value_base_not_iva = value / 1.15
// ej: $50 / 1.15 = $43.48

reference: Tu identificador unico de orden. Maximo 40 caracteres. Lo recibiras de vuelta en el webhook.

url_back: MedianetPay hace un POST aqui cuando el pago termina. Debe ser una URL publica accesible desde internet.

webhookWebhook (url_back)

Recibir confirmacion de pago

Cuando el cliente termina el pago (o lo cancela), MedianetPay hace un POST a tu url_back con el resultado. Este endpoint debe ser publico y responder 200 OK.

check_circle response: "Aprobada" — Pago exitoso. Acredita la orden.
cancel response: "Rechazada" — Pago fallido. No procesar.
info Siempre verifica el estado via webhook, no por la URL de retorno del cliente. El cliente puede cerrar el navegador antes de que lo redirijan.
security Seguridad: filtra las IPs entrantes a las de MedianetPay. Guarda el reference original para vincular con tu orden.
POST a tu url_back (JSON de MedianetPay) 
{
  "response":          "Aprobada",
  "reference":         "ORD-2024-001",
  "currency":          "USD",
  "amount":            "50.00",
  "method":            "VISA",
  "payment_reference": "MEDIANET-PAY-00123",
  "authorization":     "789012",
  "deferred":          null,
  "deferred_term":     null,
  "additional_data":   {}
}
Handler en Python (FastAPI) 
@app.post("/webhook/pago")
async def webhook_medianet(request: Request):
    data = await request.json()

    reference = data["reference"]      # tu ID de orden
    aprobada  = data["response"] == "Aprobada"
    auth_code = data.get("authorization")

    if aprobada:
        await db.marcar_pagada(reference, auth_code)
    else:
        await db.marcar_fallida(reference)

    return {"ok": True}  # siempre 200

Campos del request

Campo Tipo Max Descripcion
username String 30 Correo de acceso a MedianetPay requerido
key_webservice String 36 Llave de autenticacion de la API requerido
reference String 40 ID unico de tu orden. Lo recibiras en el webhook requerido
currency String 3 Solo "USD" requerido
value Numerico — Monto total con IVA incluido requerido
value_base_not_iva Numerico — Monto sin IVA. Formula: value / 1.15. Puede ser 0 requerido
iva Numerico 99 Porcentaje de IVA. Ecuador: 15 requerido
description String 200 Descripcion visible para el cliente en la pagina de pago requerido
url_back URL 500 Endpoint de tu servidor que recibe el resultado (POST JSON). Debe ser publico requerido
url_redirect URL 500 URL a la que el cliente es redirigido al terminar el pago requerido
person_data Objeto — Datos del comprador (mejora aprobacion) opcional
person_name String — Nombre del comprador opcional
person_lastname String — Apellido del comprador opcional
person_email String — Email del comprador opcional
person_document String — Cedula, RUC o pasaporte opcional
person_document_type String — CC Cedula · CE Ext. · PP Pasaporte · RUC opcional
errorManejo de errores

Errores comunes y como resolverlos

Credenciales invalidas

El username o key_webservice son incorrectos o la llave fue regenerada. Verifica en el portal y regenera si es necesario.

SSL no valido

Tu url_back o url_redirect deben usar HTTPS con un certificado SSL valido (no autofirmado). MedianetPay no entrega a HTTP ni SSL invalido.

reference duplicada

La reference ya fue usada. Cada pago debe tener una referencia unica. Agrega timestamp o UUID si es necesario.

Timeout del request

Configura timeout de al menos 30 segundos al llamar a la API. En horas pico puede tardar mas. Implementa reintentos con backoff exponencial.

Manejo robusto de errores (Python) 
import requests

def crear_pago(payload: dict) -> str:
    try:
        res = requests.post(
            URL,
            json=payload,
            timeout=30
        )
        res.raise_for_status()

        data = res.json()

        if "link" not in data:
            raise ValueError(
                data.get("message", "Respuesta inesperada")
            )

        return data["link"]

    except requests.exceptions.Timeout:
        raise RuntimeError("API no responde. Reintentar.")

    except requests.exceptions.HTTPError as e:
        raise RuntimeError(f"Error HTTP: {e.response.status_code}")

    except Exception as e:
        logger.error(f"Error creando pago: {e}")
        raise
